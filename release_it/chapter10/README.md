## 📖 Chapter Summary from the Book

This chapter presents a case study of what happens when traffic spikes **7.5x normal** in just 8 minutes — an "Eight-Minute Hour" where an hour's worth of traffic arrives in minutes. The key insight: **autoscaling cannot react fast enough**.

### Key Problems Identified:
1. No load shedding mechanism
2. Poor capacity planning (designed for average, not peak)
3. Synchronous dependencies causing cascading timeouts
4. Retry storms amplifying load

### Key Solutions:
- Design for peak +20% capacity
- Implement graceful degradation
- Add load shedding (reject excess early)
- Handle retries with exponential backoff + jitter
